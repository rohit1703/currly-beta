  Option A — Popular past queries

  This is exactly what Google does. Requires a search_queries table to log
  searches.

  Pre-define a list of intents like "automate invoices", "edit videos", "write
  code", "generate images" and fuzzy-match them against what the user types.
  Fast, no data collection needed, but limited to what you manually define.

  Option C — Tool description snippets
  Search tool descriptions too — so typing "automate" also surfaces tools whose
  description mentions automation, and you show the tool name as a suggestion.
  Broader but noisier.

  ---
  My recommendation: A + 1 — log every search query, and combine popular past
  searches with tool name matches. It starts empty but gets smarter the more
  people use it. This is also useful data for you to see what users are actually
   searching for.
