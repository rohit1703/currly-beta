  There are two types of suggestions Google actually shows:

  1. Tool name completion — "git" → "GitHub Copilot", "Gitness" (what we have

  2. Intent/query completion — "automate invoices" → "automate invoices for
  freelancers", "automate invoice reminders" (what you're describing)

  For #2, the options are:

  Option A — Popular past queries
  Store searches users actually type, surface the most common ones as
  suggestions. "automate invoices" shows up because 50 other users searched it.
  This is exactly what Google does. Requires a search_queries table to log
  searches.

  Option B — Category + use-case suggestions
  Pre-define a list of intents like "automate invoices", "edit videos", "write
  code", "generate images" and fuzzy-match them against what the user types.
  Fast, no data collection needed, but limited to what you manually define.

  Option C — Tool description snippets
  Search tool descriptions too — so typing "automate" also surfaces tools whose
  description mentions automation, and you show the tool name as a suggestion.
  Broader but noisier.

  ---
  My recommendation: A + 1 — log every search query, and combine popular past
  searches with tool name matches. It starts empty but gets smarter the more
  people use it. This is also useful data for you to see what users are actually
   searching for.

⏺ Spelling correction works best at the database level using PostgreSQL's
  pg_trgm extension — it measures how similar two strings are, so "coidng"
  scores high against "coding".

  The pipeline will be:
  1. FTS → results? return them

  3. NEW: Trigram fuzzy match → last resort, catches typos

  gin_trgm_ops);
  create index if not exists tools_description_trgm_idx on tools using
  gin(description gin_trgm_ops);

  -- Fuzzy search RPC function
  create or replace function fuzzy_search_tools(search_query text, match_count
  int default 20)
  returns table (
    id uuid,
    name text,
    slug text,
    description text,
    main_category text,
    pricing_model text,
    image_url text,

      id, name, slug, description, main_category,
      pricing_model, image_url, is_india_based, website, launch_date
    from tools
    where name % search_query
       or description % search_query
    order by greatest(
      similarity(name, search_query),
      similarity(description, search_query)
    ) desc
